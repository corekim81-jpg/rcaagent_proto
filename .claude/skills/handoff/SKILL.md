# 마무리 (handoff.md 기록 → to-do.md 갱신 → 커밋/푸시)

세션이 끝날 때 다음 세션(사람이든 Claude든)이 처음부터 다시 파악하지 않아도 되도록,
방금까지 한 일과 다음에 할 일을 저장소에 남긴다.

## 진행 순서

1. **이번 세션에서 한 일을 정리한다.** 대화 내용을 훑어서 실제로 완료된 작업
   (커밋/PR로 이어진 것, 고친 버그, 내린 결정)을 간단히 요약한다. 아직 안 끝난 것과
   섞이지 않게 "완료"와 "다음에 할 일"을 구분한다.

2. **`handoff.md`에 새 항목을 추가한다.** 기존 항목 위에 쌓는 형식(가장 최근이 위)을
   유지하고, 이번 세션에서 뭘 했는지 + 다음에 뭘 하기로 했는지를 적는다. 추가한 뒤
   전체 항목 수가 10개를 넘으면, 가장 오래된 항목부터 지운다 — 단, 지우기 전에 계속
   참고할 가치가 있는 내용(중요한 결정, 반복될 만한 문제)이 있으면 먼저
   `changelog.md`나 `known-pitfalls.md`로 옮긴 뒤 지운다.

3. **`to-do.md`를 갱신한다.** 이번 세션에서 끝난 항목은 지우고, 새로 남은 일이나
   다음에 이어서 할 일은 태스크로 추가한다. `plan.md`의 큰 그림과 어긋나는 항목이
   보이면 사용자에게 짚어준다.

4. **커밋하고 푸시한다.** `handoff.md`, `to-do.md`(및 관련해서 옮겼다면
   `changelog.md`/`known-pitfalls.md`)만 스테이징해서, "세션 마무리: <한 줄 요약>"
   같은 명확한 메시지로 커밋한다. 지금 브랜치가 `main`이면 새 브랜치를 따서 커밋하고
   PR을 연다 (`main` 직접 push 금지, `CLAUDE.md` 규칙). feature 브랜치 위에서 작업
   중이었다면 그 브랜치에 그대로 커밋해도 된다.

5. **결과를 짧게 보고한다.** 무엇을 기록했는지, 커밋/PR이 어디로 갔는지 한두 줄로
   알려준다.

## 참고

- 이 스킬은 문서 갱신 + 커밋/푸시(+필요시 PR)까지만 한다. 미완성 기능을 억지로
  마무리 짓거나 테스트 없이 커밋하지 않는다 — 하다 만 작업은 그대로 "다음에 할 일"로
  남기는 게 맞다.
- `handoff.md`/`to-do.md` 갱신은 코드 변경이 아니므로 `ruff`/`pytest`를 다시 돌릴
  필요는 없지만, 세션 중 코드도 같이 바뀐 상태라면 커밋 전에 `CLAUDE.md`의 필수 체크
  (`ruff check .`, `pytest`)를 먼저 통과시킨다.
