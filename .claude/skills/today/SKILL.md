# 오늘 시작 (git pull → handoff.md 확인 → to-do.md 정리)

세션마다 어제 뭘 했는지, 오늘 뭘 해야 하는지 사람이 매번 설명하지 않아도 되도록,
저장소에 이미 있는 문서(`handoff.md`, `to-do.md`)를 읽어서 브리핑한다.

## 진행 순서

1. **`git pull`.** 다른 세션/사람이 머지한 내용을 먼저 받는다. 지금 브랜치가 어제
   작업하던 feature 브랜치라면 그대로 두고, `main`이면 최신 상태인지만 확인한다.
   충돌이나 뒤처짐이 있으면 사용자에게 먼저 알린다.

2. **`handoff.md`를 읽는다.** 가장 최근 항목(들)을 보고 최근 세션에서 무엇을 했는지,
   다음에 뭘 하기로 했었는지 파악한다. 파일이 없거나 비어있으면 그냥 넘어간다.

3. **`to-do.md`를 읽는다.** 지금 열려있는 항목들을 정리해서 보여준다. `handoff.md`에서
   파악한 맥락과 연결해서 "어제 이거 하다 말았고, 오늘은 이걸 이어서 하면 됩니다"
   식으로 설명한다.

4. **간단히 브리핑하고 사용자 지시를 기다린다.** 이 스킬 혼자서 다음 작업을 마음대로
   시작하지 않는다 — 오늘 뭘 할지 사용자가 정할 수 있게 상황만 정리해서 보여주는
   것이 목적이다. `known-pitfalls.md`에 최근에 자주 겪은 문제가 있으면 관련 작업을
   시작하기 전에 한 줄로 짚어준다.

## 참고

- `plan.md`는 거의 안 바뀌는 큰 그림이라 매번 다시 읽을 필요는 없다. 사용자가 방향을
  헷갈려하거나 오랜만에 돌아온 것 같으면 그때 같이 요약해준다.
- 이 스킬은 읽기 전용이다 — 파일을 고치거나 커밋하지 않는다. 세션을 마무리할 때
  기록을 남기는 건 `handoff` 스킬의 역할이다.
